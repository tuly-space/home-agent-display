# 家庭 AI Agent 显示屏
## 从 Skylight / DAKboard 看软硬结合新机会

> 版本：2026-03-10  
> 目的：从 Skylight、DAKboard、Cozyla 三条路线出发，分析家庭显示屏赛道现状，论证 AI agent 切入的机会，收敛到软件先行的产品路线

---

## Slide 1｜这个赛道在说什么

> 把家里的"信息中心"从冰箱贴、白板、群消息  
> 升级成一块**常亮的家庭协作屏**

核心功能：共享日历 · 任务分配 · 照片展示 · 家庭提醒

**市场信号：**
- Skylight 累计销量 150万+ 台，$50M 融资，99% YoY 增长
- Amazon 上家庭日历屏品类搜索量持续上升
- 付费意愿强：硬件 $159–$1099，订阅最高 $79/年

## Slide 5｜Skylight — 用户怎么说 & AI 现状

**正面评价：**
> *"这是我们家送出去最受欢迎的礼物，爷爷奶奶一分钟就学会了"*

> *"终于不用每天早上大声喊'今天谁有活动'了"*

**高频差评（结构性局限）：**
- 「为什么不能直接在屏幕上加事件？非要用手机 App」
- 「没有提醒推送，屏幕挂在厨房但我不在旁边」
- 「AI 功能在哪？感觉就是个更好看的 Google Calendar」

**AI 现状：** 目前几乎为零。仅有 Magic Import（OCR 解析）和食谱关键词匹配。

**本质：** Skylight 是精致的"展示工具"，不是 agent。告诉你日历里有什么，但不会主动帮你管。

## Slide 11｜赛道结构判断

### 市场验证了什么

✅ 家庭愿意为"共享信息屏"付费（Skylight 150万台是铁证）  
✅ 高端用户愿意付 $600–$1000（Cozyla 的存在）  
✅ 开放平台有社区价值（DAKboard DIY 用户群）  
✅ "解决 mental load"是真实情绪需求（Hearth 的方向）

### 市场没解决什么

❌ 没有一家真正做到 AI agent 级别的主动协调  
❌ 所有产品都是"展示工具"，等用户来看，不会主动推动  
❌ 家庭信息仍然碎片化：妈妈脑子里、群消息里、学校 PDF 里……  
❌ 高摩擦：还是要手动录入，还是要记住去看

## Slide 12｜核心洞察

### 不是"更好的日历屏"
### 是"替家庭做协调的 agent"

Skylight 证明了需求。但 AI 带来的是**根本性差异**——

从"被动展示"到"主动协调"：不只是告诉你日历里有什么，而是帮你**记住、分配、推进、跟进**。

---

## Slide 13｜目标人群：谁在承受 Mental Load

**Mental Load = 家庭运营的隐形认知负担**

不只是"做事"，而是：**记住有什么事、想着什么时候做、提醒别人做、检查有没有做完**

**1. 婚后家庭女主**
- 承担 70-80% 的家庭 mental load（研究数据）
- 痛点：「我不是不想分担，是我得先记着，然后再教他」
- 需求：有人帮我记住、帮我协调、帮我推进——不只是展示给我看

**2. 恋爱中的男人**
- 想表现体贴，但不知道从哪里切入
- 痛点：「我不是不关心，就是总忘」
- 需求：帮他主动记住，让他"显得靠谱"

**3. 有未成年子女的父母**
- 孩子日程指数级复杂（补习/兴趣班/学校活动）
- 痛点：「光是管孩子的事就已经用完我的脑子了」
- 需求：孩子的任务自动分解，进度可见，不用反复追问

## Slide 14｜我们的差异化：主动 vs 被动

| 维度 | 现有产品 | 我们 |
|---|---|---|
| 信息获取 | 手动录入 | 自动抓取（邮件/群/拍照） |
| 行动模式 | 展示，等你来看 | 主动推送 + 提醒 + 建议 |
| 任务分配 | 手动分配 | 基于成员时间/能力建议分配 |
| 冲突处理 | 没有 | 自动识别冲突并提出方案 |
| 家庭沟通 | 看同一块屏 | agent 作为协调中间人 |

## Slide 15｜商业化思路：入口级机会

家庭显示屏是**进入家庭场景的硬件入口**，背后连接的是家庭个性化服务、数据私有化、以及未来 to-agent 的服务生态。

### 三层收入模型

| 层次 | 模式 | 逻辑 |
|---|---|---|
| 1 | **硬件销售** | 一次性获客，覆盖成本，建立物理触点 |
| 2 | **订阅制（卖 Token）** | AI agent 能力按用量计费；家庭个性化、学习、主动协调消耗 token，持续复购 |
| 3 | **未来服务抽成** | 家庭场景内的 to-agent 服务（购物/预约/教育/健康）从平台走，抽取佣金 |

### 两条切入路径

**路径 A · Skylight 模式（App 先行）**
- 先做手机 App，验证家庭协调 agent 核心功能
- 积累用户和数据后，推出配套硬件屏
- 硬件作为"升级体验"，不是前提

**路径 B · DAKboard 模式（Agent OS + 社区）**
- 学 MIUI 成长路径：做开放的家庭 Agent OS，兼容任意屏幕
- 社区化运营：DIY 玩家 → 口碑传播 → 大众用户
- OS 生态形成后，推官方认证硬件

**核心护城河：数据私有化 + 持续学习**  
家庭数据不上云、agent 越用越懂你家——这是大厂做不到的。

## Slide 16｜产品路线：软件先行，三步走

### 短期（0–12个月）· 家庭智能体 App

核心从 Calendar 入手，做家庭运营助理而非更好的日历：
- 接入 Google/Apple/Outlook 日历
- 解析群消息、邮件、学校通知（照片识别）
- AI 主动生成每日/每周家庭简报
- 冲突提醒 + 任务分配建议
- "这件事谁来负责" → 一键指派 + 自动跟进

**营销角度：** 不卖"日历"，卖"解放妈妈的大脑"

### 中期（12–24个月）· 社区 + 硬件兼容

- 支持旧 iPad / 安卓平板 / 树莓派跑我们的 App
- 开放 DIY 社区（类 DAKboard 但有 AI 加持）
- 适配市场上主流显示屏

### 长期（24个月+）· 自然生长出硬件

- 开放硬件规格（类 DAKboard 但有标准）
- 官方推荐机型
- 视情况推出自有品牌屏

**核心原则：** 硬件不是目的，是软件社区成熟后的自然延伸

## Slide 17｜从 Skylight 的差评里找机会

Skylight 最高频的差评，就是我们的 PRD：

| Skylight 差评 | 我们的回应 |
|---|---|
| 「要用手机 App 才能加事件，为什么不能在屏上？」| 屏幕支持直接触控编辑 + 语音添加 |
| 「没有提醒推送，我不在旁边就错过了」 | 主动推送到手机 + 家庭成员广播 |
| 「AI 功能在哪？感觉就是个日历」 | AI 就是核心：主动提醒、分配、协调 |
| 「录入太麻烦，学校发的东西要手动敲」 | 自动解析：拍照/邮件/群消息全吃进来 |
| 「家里人各管各的，还是没有协调起来」 | agent 主动协调，不是被动展示 |

## Slide 18｜总结

1. **市场已被验证**：Skylight 150万台证明了需求，但没人做到 AI agent
2. **时机对了**：LLM 能力在 2024-2025 才到了能做实用家庭 agent 的门槛
3. **切入点清晰**：软件先行，从 Calendar + Mental Load 入手，目标用户是家庭女主/父母
4. **差异化明确**：主动 agent vs 被动展示，不只是"更好的日历"
5. **路线合理**：软件 → 社区 → 兼容硬件 → 自有硬件，每步有验证节点

> **"不是展示家庭信息的屏幕，是替家庭做协调的 agent"**
